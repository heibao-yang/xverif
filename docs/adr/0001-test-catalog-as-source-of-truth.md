# 以测试目录和能力清单作为测试体系的唯一事实源

xverif 的测试身份、层级、能力、环境要求和门禁归属统一由一个测试目录/能力清单定义，Makefile、pytest 与回归脚本只负责消费或编排该清单。测试代码和 fixture 可以继续靠近所属组件存放，不以物理迁移到单一 `tests/` 目录作为统一测试体系的前提；这样既消除多套入口各自定义分层造成的漂移，也避免高风险、低收益的批量路径迁移。

## Considered Options

- 选择：统一事实源，保留测试与 fixture 的合理就近布局。
- 拒绝：先把全部测试和 fixture 物理迁移到统一目录；迁移面大，但不能自动解决分层和门禁漂移。
- 拒绝：只整理 Makefile target 与脚本别名；改动较小，但仍保留多套隐式测试清单。

## Consequences

- 每个测试必须能映射到唯一测试身份，并声明层级、能力、环境要求、成本和门禁归属。
- 新入口不得维护独立测试文件列表；临时选择集也必须通过统一清单表达。
- 允许组件内测试和复杂 fixture 保持原路径，但路径不再决定其测试语义。
